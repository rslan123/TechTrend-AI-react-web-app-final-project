# TechTrend AI — תיעוד פרויקט

פרויקט גמר בקורס פיתוח אפליקציות מובייל — מכללת רופין 2026

## מה האפליקציה עושה

TechTrend AI היא אפליקציית ניתוח שוק המניות המשתמשת בלמידת מכונה כדי לייצר איתותי כיוון שעתיים (קנה/מכור/החזק) עבור כל מניה במדד S&P 500. כל תחזית מגיעה עם ציון דיוק מבוסס אימות כך שהמשתמש תמיד יודע האם האיתות הרוויח את הסיווג שלו — או לא.

הרעיון המרכזי: המודל מסרב לתת איתות קנה/מכור אם דיוק האימות שלו נמוך מ-53%, ומחזיר NO_EDGE במקום. זה מה שהופך אותו למעניין טכנית מעבר לאפליקציית CRUD רגילה.

## איך האפליקציה עובדת — רמה גבוהה

TechTrend AI היא אפליקציה תלת-שכבתית:

1. **React Frontend** — מה שהמשתמש רואה ומשתמש בו
2. **Node/Express Backend** — המתווך, מקבל בקשות מ-React ושולח אותן ל-Python
3. **Python ML Engine** — עושה את עבודת החיזוי בפועל

כאשר משתמש מבקש תחזית:
```
React שולח בקשה ל-Node
  → Node מפעיל Python כתהליך נפרד
    → Python מוריד נתונים מ-Yahoo Finance
    → Python מאמן מודל XGBoost ומאמת אותו
    → Python מדפיס תוצאה בפורמט pipe-delimited
  → Node לוכד את הפלט ושולח ל-React כ-JSON
→ React מפרסר את המחרוזת ומרנדר גרפים וכרטיסים
```

נתוני רשימת המעקב נשמרים ב-SQLite על השרת. כל תחזית נרשמת אוטומטית לקובץ CSV.

### App.jsx — המעטפת

זו לא עמוד, זו המיכל שבתוכו הכל חי. כאשר האפליקציה נטענת, `main.jsx` מרנדר את `App` עטוף ב-`BrowserRouter`. React Router מיירט את כל שינויי ה-URL כך שניווט לעולם לא מפעיל טעינה מחדש של הדף — הוא פשוט מחליף איזה קומפוננט מרונדר בתוך `<Routes>`.

הניווט משתמש ב-`useLocation()` לקבלת נתיב ה-URL הנוכחי ומשווה אותו לפרופ `to` של כל קישור כדי להחליט מי מקבל את הסגנון הכחול הפעיל. קומפוננט `PageWrapper` עוטף את כל הנתיבים ומשתמש ב-pathname כ-React `key` — כאשר המפתח משתנה React מבטל ומרנדר מחדש את ה-div, מה שמפעיל מחדש את אנימציית ה-CSS `fadeSlideIn` ומעניק את מעבר הדף החלק.

### עמוד הבית

**זרימת משתמש:** נחיתה בעמוד ← פס השוק נטען ← גיבור גלוי מיד ← כרטיסי תכונות גלויים ← איתותים אחרונים נטענים מרשימת המעקב

**זרימת קוד:**

`STRIP_TICKERS` הוא מערך סטטי. הדף מרנדר `StripCard` אחד לכל טיקר. כל `StripCard` הוא קומפוננט עצמאי עם `useState` ו-`useEffect` משלו. האפקט מופעל בטעינה, קורא ל-`/api/predict/NVDA?source=auto`, ממתין לתגובה, מפרסר את מחרוזת הפייפ, וקורא ל-`setData()`. עד שהנתונים מגיעים הכרטיס מציג אנימציות skeleton pulse. לאחר הגעת הנתונים הוא מציג sparkline (Recharts LineChart ללא צירים), מחיר חי, שינוי אחוז בין-שעתי, ותג verdict.

`RecentSignals` יש לו `useEffect` משלו שקורא ל-`/api/watchlist` פעם אחת בטעינה, לוקח את 4 הרשומות האחרונות, הופך אותן כך שהחדשה ביותר ראשונה, ומרנדר אותן ككרטיסים הניתנים ללחיצה המקשרים לעמוד התחזית.

שום דבר בדף הזה לא מחכה לדבר אחר — כל הבקשות עצמאיות וממשק המשתמש מתמלא בהדרגה כשהתוצאות מגיעות.

### עמוד התחזית

**זרימת משתמש:** הקלד טיקר או בחר מהשלמה אוטומטית ← לחץ על "חזה" ← skeleton טעינה ← תוצאות מופיעות עם גרף, סטטיסטיקות, פס RSI

**זרימת קוד:**

לשדה החיפוש יש handler של `onChange` שקורא ל-`handleInput`. זה מסנן את `SP500_TICKERS` (מערך סטטי של 500 רשומות ב-`sp500tickers.js`) מול מה שהמשתמש הקליד, תואם גם התחלת סימבול וגם הכללת שם חברה, ושם עד 7 התאמות ב-state של `suggestions`. הרשימה הנפתחת מרנדרת מה-state הזה. `useRef` על div המיכל ו-`useEffect` שמוסיף listener של `mousedown` למסמך מטפל בסגירת הרשימה הנפתחת בלחיצה מחוצה לה.

כאשר לוחצים על "חזה", `runPrediction` מופעל. הוא מגדיר `loading: true`, `data: null`, `error: null`. הוא קורא ל-`/api/predict/TICKER`, מקבל חזרה `{ raw: "RESULT|AAPL|182.34|BUY|..." }`. הוא מפצל לפי `|` וממפה כל אינדקס לשדה בעל שם. זמנים, מחירים ו-SMA הם מחרוזות משנה מופרדות בפסיק שמפוצלות שוב למערכים. אלה הופכים למערך `history` — 20 אובייקטים כל אחד עם `label`, `price`, ו-`sma`.

`setData(parsed)` מפעיל רינדור מחדש. ה-verdict קובע את `lineColor` דרך מפת `VERDICT_COLOR`. הגרף הוא `ComposedChart` עם `Area` למילוי הגרדיאנט (ללא stroke, רק הגרדיאנט המוגדר ב-`<defs>`) ו-`Line` לקו המחיר בפועל הצבוע לפי verdict. `Line` מקווקו שני מציג SMA-20. פס RSI הוא div רגיל עם `width: ${rsi}%` כסגנון inline, צבוע לפי סף.

### עמוד ההשוואה

**זרימת משתמש:** בחר שני טיקרים (או לחץ על preset) ← שניהם נטענים בנפרד ← באנר מנצח ← סטטיסטיקות זה לצד זה ← גרף שכבות ← טבלת השוואה

**זרימת קוד:**

ה-state מוכפל — `leftData/rightData`, `leftLoading/rightLoading`, `leftError/rightError`. `fetchSide` היא פונקציה גנרית שמקבלת קבוצת setters כפרמטרים, כך שאותה פונקציה מטפלת בשני הצדדים. כאשר לוחצים על preset, `fetchSide` נקראת פעמיים עם setters שונים. כל אחת מסתיימת בנפרד — אם אחת איטית השנייה עדיין מציגה.

`buildOverlayData` ממזגת את שתי ההיסטוריות של 20 עמודות לפי אינדקס מערך למערך אחד: `[{ i, label, leftPrice, rightPrice }, ...]`. ה-`ComposedChart` מרנדר שני Areas עם גרדיאנט ושני Lines על אותו ציר Y. שני הקווים חולקים את אותו גרף כך שמחיריהם ניתנים להשוואה ישירה ויזואלית.

באנר המנצח מנקד כל טיקר: BUY=2, HOLD=1, NO_EDGE=0, SELL=-1, ואז מוסיף `cv_accuracy / 100` כמכריע. ציון גבוה יותר מנצח. אם הציונים בטווח של 0.01 זה מזה מציג TIED.

קומפוננט `CompareRow` בטבלת ה-head-to-head מקבל `higherIsBetter` כ-prop — עבור ביטחון ודיוק backtest, ערך גבוה יותר עדיף לכן הוא מדגיש את המספר הגדול יותר בירוק.

### עמוד רשימת המעקב

**זרימת משתמש:** הדף נטען ← טיקרים שמורים נטענים מה-DB ← בקשת מחיר חי מופעלת לכל אחד ← כרטיסים מתמלאים עם sparklines ודלתא מחיר

**זרימת קוד:**

שתי מבני נתונים נפרדים — `saved` (שורות DB גולמיות מ-`/api/watchlist`) ו-`live` (מפה של טיקר ← תוצאת תחזית מפורסרת). שני מצבי טעינה נפרדים — `loadingDb` לבקשת ה-DB הראשונית, ו-`loadingLive` שהוא אובייקט עם מפתח לפי טיקר כך שכל כרטיס יודע את מצב הטעינה שלו בנפרד.

`useEffect` בטעינה קורא ל-`fetchWatchlist`. `useEffect` שני עוקב אחר מערך `saved` — כאשר הוא מתמלא, הוא קורא ל-`refreshAll` שלולאת דרך טיקרים וקוראת ל-`fetchLive` לכל אחד ברצף באמצעות `await` בתוך לולאת `for...of`. זהו ה-staggering המכוון — לא `Promise.all` שהיה מפעיל הכל בבת אחת.

כל כרטיס מציג את המחיר השמור משורת ה-DB (מה שהיה כאשר הוספת אותו) ואת המחיר החי מתוצאת התחזית. `PriceDelta` מחשב `((live - saved) / saved) * 100` ומציג בירוק או אדום. ה-sparkline הוא `LineChart` רגיל ללא צירים — רק צורת תנועת המחיר צבועה לפי verdict.

מחיקה קוראת ל-`/api/watchlist/TICKER`, ואז מסירה את הרשומה מה-state של `saved` וגם של `live` מקומית כך שהכרטיס נעלם מיד ללא צורך ב-refetch.

### עמוד החינוך (מדרשת השוק)

**זרימת משתמש:** ראה 6 כרטיסי מושגים ← לחץ על אחד להרחבה ← קרא הסבר, התנסה עם גרף או סליידר ← לחץ על אחר להחלפה

**זרימת קוד:**

ה-state `active` מחזיק את ה-ID של הכרטיס הפתוח כרגע או `null`. `ConceptCard` משווה `active === id` להחלטה האם לרנדר את ילדיו. לחיצה על הכותרת קוראת ל-`onToggle` — אם כבר פתוח מעבירה `null` (סגור), אם סגור מעבירה את ה-id (פתח). רק אחד יכול להיות פתוח בכל זמן כי יש ערך `active` אחד בלבד.

תוכן כל מושג הוא קומפוננט משלו — `SMAContent`, `RSIContent` וכו'. זה שומר את הקובץ מאורגן ואומר שלכל אחד יש state מבודד משלו לסליידרים.

ל-`SMAContent` יש `const [period, setPeriod] = useState(20)`. מערך המחירים נוצר פעם אחת עם `useMemo([], [])` — מערך תלות ריק אומר שהוא לעולם לא נוצר מחדש. ערכי ה-SMA משתמשים ב-`useMemo([prices, period])` — הם מחושבים מחדש רק כאשר `period` משתנה. גרירת הסליידר מעדכנת את ה-state של `period` ← מפעיל חישוב useMemo מחדש ← Recharts מרנדר מחדש עם קו SMA חדש. קו המחיר נשאר זהה.

`BollingerContent` עובד באותו אופן עם סליידר `mult` (מכפיל).

`RSIContent` משתמש בשני גרפים מוערמים — `LineChart` מחיר רגיל למעלה, `ComposedChart` עם קו RSI וקווי ייחוס ב-30 וב-70 למטה. `ReferenceLine` הוא קומפוננט Recharts שמצייר קו אופקי בערך Y קבוע.

### עמוד לוג האיתותים

**זרימת משתמש:** הדף נטען ← כל התחזיות מוצגות החדשה ביותר ראשונה ← סנן לפי verdict או מקור ← חפש לפי טיקר ← מיין עמודה כלשהי ← ייצא או נקה

**זרימת קוד:**

`fetchLog` קורא ל-`/api/log` שקורא את ה-CSV בשרת ומחזיר מערך JSON. זה נקרא בטעינה דרך `useEffect` וגם מחובר לכפתור Refresh.

`filtered` הוא `useMemo` שתלוי ב-`[rows, verdictFilter, sourceFilter, search, sortCol, sortDir]`. בכל פעם שמסנן משתנה, הוא מחשב מחדש: מסנן לפי verdict, מסנן לפי מקור, מסנן לפי חיפוש טיקר, ואז ממיין לפי העמודה הפעילה. המיון מטפל בעמודות מחרוזת וגם מספריות על ידי בדיקת `typeof`.

`useEffect` שני עוקב אחר ערכי הסינון/חיפוש/מיון ומאפס את `page` ל-1 בכל פעם שהם משתנים — כדי שלא תגיע לדף 3 של תוצאה מסוננת שיש בה רק דף 1.

Pagination מחלק את `filtered` לחתיכות של `PAGE_SIZE = 15`. `pageRows` הוא פשוט `filtered.slice((page-1)*15, page*15)`.

פונקציית הייצוא יוצרת מחרוזת CSV ממערך `filtered` הנוכחי (לא כל השורות — מה שאתה רואה זה מה שאתה מייצא), עוטפת אותה ב-`Blob`, יוצרת URL אובייקט זמני, לוחצת בתוכנה על אלמנט עוגן נסתר להפעלת ההורדה, ואז מבטלת את ה-URL לשחרור זיכרון.

נקה קורא ל-`DELETE /api/log` שמריץ `fs.unlinkSync` בשרת למחיקת הקובץ, ואז `setRows([])` מקומית כך שהטבלה מתרוקנת מיד.

## מנוע ML — Python

**8 features מהנדסים מנתוני מחיר גולמיים:**

1. SMA-20 — ממוצע נע פשוט של 20 תקופות
2. Price vs SMA — מרחק מנורמל של מחיר מ-SMA
3. RSI-14 — מדד כוח יחסי, מדד מומנטום
